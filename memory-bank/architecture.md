## UX Laws Applied
- **Hick's Law** → 5 sidebar items max (was 9)
- **Miller's Law** → 7±2 chunks per view
- **Jakob's Law** → Sidebar like Linear/Notion
- **Von Restorff** → AI Passport visually distinct
- **Pareto 80/20** → Passport + Registry = core
- **Occam's Razor** → Remove everything non-essential
- **Peak-End Rule** → Hero = trust peak, not sales pitch
- **Fitts' Law** → One large CTA per screen
- **Gestalt Proximity** → Bento grid groups related info
- **Aesthetic-Usability** → Beautiful = more trustworthy
- **Serial Position** → First item (Explore) + last (Verify) most remembered
- **Don Norman Signifiers** → Color = action only
- **Postel's Law** → Search accepts flexible input

## NN/g 2026 Applied
- UI is no longer differentiator → focus on deep UX
- Trust is #1 design problem for AI → trust-first hero
- AI slop backlash → no fake social proof
- MX (Machine Experience) → semantic HTML for AI agents

## Apple HIG Applied
- Sidebar for apps with many items
- Minimum items required → reduced to 5
- Color reserved for interactive elements
- Predictable layouts, controls where expected
